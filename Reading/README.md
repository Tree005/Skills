# 阅读指导.Skill (Reading Guide)

## 技能概述

一个"读书搭档"风格的阅读指导技能，核心目标不是帮你分析书，而是陪你一起**独立思考**。

提供三种模式，根据你的需求自动切换：

| 模式 | 触发方式 | 定位 |
|---|---|---|
| 💬 读书搭档 | 发一句话/引用原文 | 对话式精读，先聊再深挖 |
| 🔍 全书概览 | "这本书讲了什么""帮我梳理" | 一句话总结 + 核心主题 + 章节脉络 |
| 🌱 消化输出 | "帮我消化""有什么启发" | 连接生活 → 提炼行动 → 批判反思 |

## 核心哲学：独立思考

> 读书不是为了找到"正确答案"，而是为了学会独立思考。

贯穿所有模式的底层原则：
- **不替用户下定论**——每个观点后面留"但是"的空间
- **鼓励批判性思考**——不让用户因为一本书写得好就全盘接受
- **帮助识别局限**——主动指出作者的偏见、时代局限、论证漏洞
- **拥抱矛盾和重塑**——两本书的观点矛盾时，不急着调和，引导用户自己思考

核心提醒：不要让任何一本书、任何一个理论、任何一句金句成为思维的终点。

## 各文件说明

### SKILL.md — 大脑

定义整个技能的行为规则：
- **模式判断规则**：根据用户输入自动选择进入哪个模式
- **💬 读书搭档模式**：对话式精读，先给解读，问一个引导性问题，等用户回应再深入
- **🔍 全书概览模式**：一句话总结 + 核心主题 + 章节脉络 + 关键人物 + 阅读建议
- **🌱 消化输出模式**：费曼技巧解释概念 → 连接用户生活 → 提炼行动项 → 批判性反思
- **独立思考原则**：不替用户下定论，鼓励质疑，识别局限
- **模式切换机制**：对话中随时可以切换

### references/note_template.md — 笔记格式

三种模板对应三种模式：

| 模板 | 触发 | 文件名示例 | 内容 |
|---|---|---|---|
| 精读笔记 | 聊深了，用户说"记下来" | `天命与选择.md` | 精读解读 + 关键发现 + 生活连接 |
| 概览笔记 | 用户说"帮我梳理这本书" | `书名_概览.md` | 一句话总结 + 核心主题 + 章节脉络 + 阅读建议 |
| 消化笔记 | 用户说"帮我消化" | `概念名_消化.md` | 大白话解释 + 生活连接 + 行动项 + 批判性反思 |

另有一个索引文件模板（`书名.md`），统一链接所有笔记。

### references/analysis_framework.md — 分析深度开关

两层分析，默认轻量，按需加载：

- **第一层：轻量解读**（默认）
  - 2-4 段自然语言解读，不套模板、不列维度
  - 一个引导性问题，等用户回应
  - 语气像聊天，不像写论文

- **第二层：完整分析**（用户说"深入分析"才触发）
  - 字面含义 → 四维度分析（主题/人物/情节/哲学）→ 原著语境 → 生活智慧 → 应用方案

## 笔记存储结构

将笔记保存路径配置为你自己的阅读笔记文件夹，例如：

```
<你的阅读笔记路径>\
└── <书名>/
    ├── <主题1>.md              ← 精读笔记
    ├── <主题2>.md
    ├── <概念1>_消化.md          ← 消化笔记
    ├── <书名>_概览.md           ← 全书概览（每本书一份）
    └── <书名>.md               ← 索引文件（链接所有笔记）
```

> **安装说明**：在 SKILL.md 中将所有 `<你的阅读笔记路径>` 替换为你实际的阅读笔记文件夹路径。

## 安装（3 步）

### 第 1 步：复制文件

将整个 `reading-guide` 文件夹复制到 `~/.workbuddy/skills/` 下：

```
~/.workbuddy/skills/reading-guide/
├── SKILL.md
├── README.md
└── references/
    ├── note_template.md
    └── analysis_framework.md
```

### 第 2 步：替换路径

用你自己的阅读笔记文件夹路径，替换以下文件中的占位符：

| 文件 | 要替换的内容 |
|------|-------------|
| `SKILL.md` | 搜索所有 `<你的阅读笔记路径>`，替换为你的实际路径（如 `/home/user/notes/reading`） |
| `references/note_template.md` | 同上，替换 `<你的阅读笔记路径>` |

> **注意**：Windows 用户用反斜杠路径（如 `D:\Notes\阅读`），Mac/Linux 用户用正斜杠路径。

### 第 3 步：准备你的阅读文件夹

确保你的阅读笔记文件夹里，每本书有一个子文件夹，里面存放该书的相关笔记。skill 会在这个结构下自动创建和管理笔记。

```
<你的阅读笔记路径>\
└── 某某书/
    ├── 主题1.md              ← 你已有的或 skill 创建的精读笔记
    ├── 主题1_消化.md          ← 消化笔记
    ├── 某某书_概览.md         ← 全书概览
    └── 某某书.md             ← 索引文件
```

安装完成！接下来开始使用 ↓

## 使用方式

**不需要显式调用**——在阅读笔记工作区内，像跟朋友聊天一样，把书中触动你的话发过来就行。

skill 会根据你说的话自动选择模式：

### 💬 想聊天/精读

随便发一句书中的原文或你的想法：

> "《被讨厌的勇气》里说'一切烦恼都来自人际关系'，你怎么看？"

AI 会给你一段轻量解读，然后问一个引导性问题。你可以顺着聊，也可以说"深入分析"让它展开。

### 🔍 想了解整本书

> "帮我梳理一下《原子习惯》"
> "这本书讲了什么？"

AI 会给出全书概览：一句话总结、核心主题、章节脉络、阅读建议。

### 🌱 想消化吸收

> "帮我消化一下这段内容"
> "读完这本书，有什么可以落地的？"
> "读完《纳瓦尔宝典》，帮我做个总结"

AI 会用大白话解释概念、连接你的生活、提炼行动项，还会引导你批判性反思——哪些认同、哪些存疑、怎么反驳。

### 💾 想记笔记

在任何模式下，说一句"记下来"或"帮我记笔记"，AI 就会按模板创建对应的笔记文件，保存在你的阅读文件夹里。

### 🔄 随时切换

对话过程中随时可以换方向：
- 聊着聊着想看全书概览 → 直接说"帮我梳理这本书"
- 看完概览想深聊某一章 → 引用那章的内容
- 聊完了想消化 → 说"帮我消化"

**不需要任何特殊指令，像说话一样自然就好。**
