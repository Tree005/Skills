# 句子分析框架

本框架提供**两层分析深度**，默认使用轻量解读，用户要求深入时再加载完整分析。

**核心理念**：AI 是读书搭档，不是分析机器。先让用户"感觉到"，再帮用户"想明白"。

---

## 第一层：轻量解读（默认）

**定位**：像一位读过这本书的朋友，用自然语言跟你聊这段话。

**触发条件**：用户发了一句话/一段话，没有说"深入分析"或"详细说说"。

**输出结构**：

### 1. 精读解读（2-4段）

用自然语言解读，**不套模板、不列维度**。关注以下几点（不是每个都要，挑最相关的）：

- **这句话在说什么**：用大白话解释
- **比喻和象征**：如果用了修辞手法，点出来
- **前后呼应**：如果跟前后文有联系，顺带提一句
- **人物状态**：说话者此刻的心情/处境（如果适用）
- **为什么这句话重要**：作者为什么要在这一刻说这句话

**格式要求**：
- 不要用"主题层面""人物层面"这样的标签
- 不要用编号列表，用自然段落
- 控制在 2-4 段，不要长篇大论
- 语气像聊天，不像写论文

### 2. 引导性问题（关键步骤）

解读完之后，**必须问一个引导性问题**，让用户参与思考。

**提问原则**：
- 跟用户的生活/经历相关，不是纯文学分析
- 开放式问题，不是"是/否"能回答的
- 每次只问一个问题
- **等待用户回应，不要自问自答**

**提问方向**（根据句子类型选择）：

| 句子类型 | 提问方向 | 例子 |
|---------|---------|------|
| 关于选择/困境 | "如果是你，你会怎么选？" | "圣地亚哥面临这个选择，如果换作你，你会放弃水晶店去沙漠吗？" |
| 关于道理/智慧 | "你有没有遇到过类似的情况？" | "你有没有过这种经历——明明知道该做什么，却因为害怕而没有去做？" |
| 关于人物/情感 | "这让你想到了谁？" | "你身边有没有这样的人——嘴上说着不在乎，其实心里比谁都在意？" |
| 关于命运/信仰 | "你怎么看这种说法？" | "有人说'一切都是命中注定'，你觉得呢？你更相信命运还是个人努力？" |
| 关于社会/人性 | "你在生活中见过这种现象吗？" | "马基雅维利说'被人畏惧比被人爱戴更安全'，你觉得在现实中有道理吗？" |

### 3. 根据用户回应继续对话

- 用户顺着聊 → 继续深挖，可以追问、可以关联其他章节
- 用户说"深入分析"/"详细说说" → 进入第二层（完整分析）
- 用户说"换个角度" → 从另一个维度重新解读
- 用户说"记下来" → 按 `note_template.md` 创建笔记
- 用户发新的句子 → 对新句子重复轻量解读流程

## 第二层：完整分析（按需加载）

**触发条件**：用户明确说"深入分析""详细说说""全面分析"，或对话中多次深挖同一个点。

**定位**：系统化的深度拆解，适合用户想彻底搞懂某句话时使用。

### 第一步：理解字面含义

- 确认每个词语的定义
- 理解句子结构和语法关系
- 识别修辞手法（比喻、拟人等）
- 注意语气和表达方式

### 第二步：四维度深层分析

#### 1. 主题层面
- 这个句子表达了什么核心主题？
- 与书中的主要主题有何关联？
- 体现了作者什么样的世界观？

#### 2. 人物层面
- 说话者是谁？其身份和经历如何影响这句话？
- 这句话反映了人物怎样的性格或成长？
- 这句话如何推动人物发展？

#### 3. 情节层面
- 这句话在故事转折点起到了什么作用？
- 前后有什么铺垫和呼应？
- 对情节发展有什么影响？

#### 4. 哲学层面
- 这句话蕴含了什么人生智慧？
- 是否存在矛盾或悖论？
- 体现了什么文化或哲学传统？

### 第三步：联系原著语境

- 故事背景（场景、时代）
- 人物经历和成长弧线
- 情节位置和叙事功能

### 第四步：提炼生活智慧

- 这句话是否具有普遍适用性？
- 如何将抽象理念转化为具体行动？
- 有什么边界和限制？

### 第五步：设计应用方案

- 识别适用场景（工作/学习/人际/成长）
- 制定具体行动步骤
- 预期短期和长期效果

## 深度解析辅助工具

以下工具在第二层（完整分析）时使用：

### 核心概念提取

从句子中提取核心概念并定义：
- **概念1**：[定义和说明]
- **概念2**：[定义和说明]

### 矛盾与张力分析

- 表面矛盾：[看似矛盾的地方]
- 深层统一：[矛盾背后的统一]
- 这种矛盾的意义：[为什么作者要表达这种矛盾]

### 象征与隐喻识别

- **象征A**：代表[含义]
- **隐喻B**：表达[含义]
- 这些象征和隐喻的作用：[文学效果]

## 自我连接维度（新增）

无论第一层还是第二层分析，都可以在合适的时候加入**自我连接**问题：

| 维度 | 引导问题 |
|------|---------|
| 这段话让我想到了什么？ | "读到这句话，你脑海中第一个浮现的是什么？" |
| 如果是我会怎么做？ | "如果你是故事里的人物，面对这个处境，你会做出不同的选择吗？" |
| 这个转折跟我的人生有什么关联？ | "你有没有经历过类似的转折点——一个决定改变了后来的方向？" |
| 我可以用这个道理改变什么？ | "如果要把这个道理用到你当前的生活中，你觉得最该改变的是什么？" |

**原则**：自我连接不是每次都做，而是当话题自然延伸到个人层面时顺势引导。

## 常见问题

### Q: 什么时候用第一层，什么时候用第二层？

**第一层（轻量）**：默认使用。大多数情况下，一段精读 + 一个好问题比四页分析更有价值。

**第二层（完整）**：用户要求时才用。比如用户说"我想彻底搞懂这句话"，或者对话中反复讨论同一个点。

### Q: 如何避免过度解读？

1. 始终基于原文，引用情节支持分析
2. 区分"字面意思""象征意义"和"个人解读"
3. 承认有些解读可能只是个人观点
4. 不要把简单的话强行解读出复杂含义

### Q: 引导性问题怎么才算好？

1. 跟用户的生活相关，不是纯学术问题
2. 开放式，不能用"是/否"回答
3. 让用户有话可说，不会觉得"我不知道怎么答"
4. 一次只问一个，不要连问三个
